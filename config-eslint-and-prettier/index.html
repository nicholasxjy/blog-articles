How to config ESLint and Prettier for your vue projects

December 02, 2018

ESlint通过对代码加以规范，尽可能的保证代码的统一性，也避免了代码中容易歧义而难以发现的bugs。Prettier 通过按照规范自动化格式代码，让代码更加易读和清晰。 本文讲述vue项目中的ESLint和Prettier的配置，希望可以帮助大家。

前言

先不关注各个文件的用处，按照下面一步一步开始配置。

1. 安装需要的依赖

1.1 安装ESLint相关依赖

你需要执行

$ npm i -D eslint babel-eslint eslint-loader eslint-plugin-vue@next

eslint-loader是webpack的loader，而eslint-plugin-vue则定义了很多vue的eslint的规则。 同时我们还需要安装eslint的prettier需要的依赖：

$ npm i -D eslint-plugin-prettier eslint-config-prettier

eslint-config-prettier 关闭了eslint和prettier可能存在的一些冲突，而 eslint-plugin-prettier 可以让prettier不符合规范的作为eslint的不规范向我们提示。

1.2 安装Prettier相关依赖

只需要执行: 

$ npm i -D prettier

2. 配置文件

2.1 ESLint的配置

ESLint的配置可以很多种形式去书写，单独的配置文件，package.json等等。我们这里单独使用配置文件，具体详情请参照Configuring ESLint

首先我们在项目里创建文件 .eslintrc，配置如下：

{
  "root": true,
  "parserOptions": {
    "parser": "babel-eslint",
    "sourceType": "module",
    "ecmaVersion": 2017
  },
  "extends": ["plugin:vue/essential"],
  "env": {
    "browser": true,
    "node": true
  },
  "plugins": ["vue"],
  "globals": {
    "CONFIG": false
  },
  "rules": {
    "vue/require-default-prop": "off",
    "vue/no-v-html": "off"
  }
}

其中主要两点是需要在extends中添加plugin:vue/essential，以及plugins中添加vue，此时我们的项目中ESLint的规则就设置完毕。

2.2 Prettier配置

同样在项目创建文件.prettierrc，配置如下：

{
  "printWidth": 80,
  "semi": false,
  "useTabs": false,
  "singleQuote": true,
  "tabWidth": 2
}

注意此时我们需要在.eslintrc中添加Prettier需要的配置，如下(前后对比)：

{

  "extends": ["plugin:vue/essential", "plugin:prettier/recommended"],
  "plugins": ["vue", "prettier"],
  "rules": {
    "prettier/prettier": "error",
    "vue/require-default-prop": "off",
    "vue/no-v-html": "off"
  }
}

此时我们的基本配置就差不多已经完成，如果你通过webpack开启你的项目，就会看到ESLint和Prettier提示的很多错误，这就需要我们解决。

3. 编辑器的配置

目前很多编辑器都有ESLint和Prettier的插件集成，所以我们完全可以通过插件，自动帮助我们实时的解决代码格式规范问题。

这里只讲一下Visual Studio Code即（VSCode）的配置：

在VSCode中你需要安装ESlint和 Prettier - Code formatter两个插件，然后在settings.json中配置如下：

{
    "prettier.eslintIntegration": true,
    "eslint.autoFixOnSave": true,
    "eslint.validate": [
        "javascript",
        "javascriptreact",
        {
        "language": "vue",
        "autoFix": true
        }
    ]
}

按照上述配置，在VS Code中当你save一个文件时，会自动帮助你解决ESLint的问题，而且会自动美化你的代码。

4. 在提交代码之前的防范措施

在我们没解决规范的问题时，我们不能就提交代码到git仓库，我们需要添加一些措施防范此种行为。

此时我们需要安装两个依赖，达到此目的：

$ npm i -D husky lint-staged

然后在我们的package.json文件中，做如下配置：

{
    "husky": {
        "hooks": {
            "pre-commit": "lint-staged"
        }
    },
    "lint-staged": {
        "*.{js,vue}": [
            "eslint --fix",
            "prettier --write",
            "git add"
        ]
    }
}
